# PatchNotes Project Status — March 13, 2026

## What PatchNotes Is
An iOS SwiftUI app — "Sleeper for video games." A social/news platform where users follow games and get curated gaming news in their feed. Uses Supabase (PostgreSQL) as backend, Cloudflare Workers for content ingestion.

## Current Architecture

### iOS App (SwiftUI)
- 4 tabs: Home (feed + game pill bar), Calendar, My Games, Esports
- Feed uses cursor-based pagination via Supabase RPCs
- Game pill bar: horizontal scroll of followed games, tap to filter feed
- Posts display with GlassCard styling, reactions, comments
- Custom `RemoteMediaImage` loader (URLSession + NSCache, not AsyncImage)
- Supabase publishable key: `sb_publishable_inRAHwLlG_tkMBM5yH0R7Q_LbpmOqJ-`

### Content Pipeline (Cloudflare Worker)
- **Worker:** `patchnotes-reddit-fetcher` on Cloudflare free tier
- **Cron:** `*/5 * * * *` (every 5 minutes)
- **Batch size:** 2 sources per invocation (~38 subrequests, under 50 limit)
- **Reddit:** Fetches via RSS/Atom feeds (JSON API returns 403 from datacenter IPs)
- **Twitter/X:** `TWITTERAPI_IO_API_KEY` secret is set on the Worker but **no active Twitter sources exist** in `bot_content_sources` table — needs accounts added to start flowing
- **AI rewriting:** Workers AI (`@cf/meta/llama-3.1-8b-instruct`) rewrites all posts
  - Filters non-gaming content (monitors, peripherals, deals)
  - Enforces complete sentences
  - Identifies which game a post is about (game-tagging)
  - Can override source-level `game_id` for cross-posted content
- **Game catalog:** Loaded once per batch (1 subrequest), 110 games currently
- **Dedup:** `bot_post_log` table prevents duplicate posts
- **Source rotation:** Processes oldest-refreshed sources first
- **No dependency on local machine** — runs 24/7 on Cloudflare's free tier

## What's Working

### Feed & Content
- Reddit RSS pipeline: fetches, AI-rewrites, filters, game-tags, inserts — runs 24/7 on Cloudflare
- ~25+ active Reddit sources across game-specific and game-agnostic bots
- Hot score formula: `ln(reactions+1)*0.6 + ln(comments+1)*0.8 + velocity*1.2 + exp(-age/259200)` with random jitter to prevent batch clustering
- Game-agnostic posts (`game_id=NULL`) appear in all users' feeds
- Game-specific posts appear only under their game pill
- Feed ordering: `hot_score DESC, created_at DESC, id DESC`

### Game Pills
- Tap targets use `onTapGesture` (not Button) to avoid ScrollView gesture conflict
- Crown/badge overlays removed — all games selectable/unselectable in Browse Games
- Game feed RPC: `fetch_game_feed_page` with `COALESCE(game_id, game_id_from_primary_badge_key)` fallback

### Bot System
- 7 specialty bots (breaking news, rumors, esports, etc.)
- ~60 per-game bots mapped to subreddits
- `source_kind='curated'` — no ugly "via X" attribution shown to users
- `is_bot` flag on profiles, `author_is_bot` in feed views/RPCs

### StoreKit 2
- Subscription view, StoreKitManager, purchase flow implemented
- Wired into app lifecycle and settings
- `errorMessage` surfaced to UI (was previously swallowed)

## Known Issues / Technical Debt

1. **RSS limitations:** No upvote/score data (can't filter by MIN_SCORE), limited media extraction (images from `<img>` tags in Atom content only)
2. **Game title matching:** Backfill migration uses `ILIKE '%title%'` which can false-match. AI game-tagging is the better long-term solution
3. **`primary_badge_key` ghost tagging:** Posts can appear in wrong game feeds via the badge key fallback even if `game_id` is corrected. Both fields need clearing for mistagged posts
4. **QA: `set_user_favorite_games` auth bypass:** RPC accepts `p_user_id` without `auth.uid()` check. Fix before production
5. **No Twitter sources active:** API key is set but no accounts in `bot_content_sources`
6. **Dragon Quest VII, other roman numeral titles:** Don't match in title backfill
7. **Cloudflare free tier limits:** 100k requests/day, 50 subrequests per invocation — currently well within limits but monitor as sources grow
